# 实现者子代理提示模板

在调度实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N：[任务名称]"
  prompt: |
    你正在实现任务 N：[任务名称]

    ## 任务描述

    [计划的完整任务文本 - 粘贴在这里，不要让子代理读取文件]

    ## 上下文

    [场景设置：这适合的位置、依赖项、架构上下文]

    ## 在你开始之前

    如果你对以下内容有疑问：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖项或假设
    - 任务描述中任何不清楚的内容

    **现在提问。** 在开始工作之前提出任何担忧。

    ## 你的工作

    一旦你清楚需求：
    1. 完全按照任务指定的内容实施
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实现有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 报告回来

    工作目录：[directory]

    **在你工作时：** 如果你遇到意外或不清楚的内容，**提问**。
    暂停并澄清总是可以的。不要猜测或假设。

    ## 在报告回来之前：自我审查

    用新的眼光审查你的工作。问自己：

    **完整性：**
    - 我完全实现了规范中的所有内容吗？
    - 我遗漏了任何需求吗？
    - 有没有我没有处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 名称是否清晰准确（匹配事物做什么，而不是如何工作）？
    - 代码是否干净且可维护？

    **纪律：**
    - 我是否避免了过度构建（YAGNI）？
    - 我只构建了请求的内容吗？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否实际验证行为（不仅仅是模拟行为）？
    - 如果需要，我是否遵循了 TDD？
    - 测试是否全面？

    如果你在自我审查中发现问题，现在修复它们，然后再报告。

    ## 报告格式

    完成时，报告：
    - 你实现的内容
    - 你测试的内容和测试结果
    - 更改的文件
    - 自我审查发现（如果有）
    - 任何问题或担忧
```
