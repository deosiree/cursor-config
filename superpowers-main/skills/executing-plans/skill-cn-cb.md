# 执行计划 (Executing Plans)

## 概览

加载计划，进行批判性审查，分批次执行任务，并在批次之间汇报以供审查。

**核心原则：** 分批执行，并设有架构师审查检查点。

**开始时宣布：** “我正在使用 executing-plans 技能来实施此计划。”

## 流程

### 第 1 步：加载并审查计划
1. 阅读计划文件
2. 批判性审查——识别对计划的任何疑问或担忧
3. 如果有担忧：在开始前向你的人类搭档提出
4. 如果无担忧：创建 TodoWrite 任务并继续

### 第 2 步：分批执行
**默认：前 3 个任务**

对于每个任务：
1. 标记为进行中 (in_progress)
2. 严格遵循每一个步骤（计划包含小粒度的步骤）
3. 按指定要求运行验证
4. 标记为已完成

### 第 3 步：报告
当批次完成时：
- 展示已实施的内容
- 展示验证输出
- 说：“准备好接收反馈。”

### 第 4 步：继续
基于反馈：
- 如有需要，应用更改
- 执行下一批次
- 重复直到完成

### 第 5 步：完成开发

当所有任务完成并验证后：
- 宣布：“我正在使用 finishing-a-development-branch 技能来完成这项工作。”
- **必需子技能：** 使用 `superpowers:finishing-a-development-branch`
- 遵循该技能以验证测试、展示选项并执行选择

## 何时停止并寻求帮助

**遇到以下情况立即停止执行：**
- 批次执行中途遇到阻碍（缺少依赖、测试失败、指令不明确）
- 计划存在导致无法开始的关键缺口
- 你不理解某条指令
- 验证反复失败

**寻求澄清而不是猜测。**

## 何时重访早期步骤

**遇到以下情况返回审查（第 1 步）：**
- 搭档根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行突破阻碍** —— 停止并询问。

## 记住
- 首先批判性地审查计划
- 严格遵循计划步骤
- 不要跳过验证
- 当计划要求时引用技能
- 批次之间：只需报告并等待
- 受阻时停止，不要猜测
