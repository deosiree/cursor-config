写一个原型设计：我已拷贝了原型设计 `prototype\importConfigV2`，需要设计它的 v2 版本，新增 JSON 配置文件导入的编辑流程。整体链路为：  
**点击「配置文件」 → 切换到子 Tab「JSON」 → 选择配置文件 → 点击「导入」 → 打开弹窗「编辑json导入词条」**。

1. **配置文件选择行为**
   1. XML 子 Tab：
      - 保持 **多选** 行为不变；
      - 下拉面板中保留 **「全选」** 与 **「清空」** 按钮。
   2. JSON 子 Tab：
      - 将配置文件选择从 **多选修改为单选**；
      - 下拉面板中 **移除「全选」和「清空」按钮**，仅支持单选 + 清除当前选择；
      - 选中的 JSON 配置文件将作为后续“导入 json”内容的来源标识。

2. **弹窗「编辑json导入词条」的目标**
   - 通过左右分栏的编辑弹窗，在不调用真实接口的前提下，体验“从 JSON 配置文件中抽取需要翻译的词条 → 编辑抽取结果 → 模拟导入”的完整交互。
   - 弹窗只在 `dataType === 'config' && configType === 'json'` 且点击「导入」时出现。
   - 左侧导入区的 Mock 内容直接取自 `gateway.json`（例如 `history_ips`、`tools` 等结构），方便以后对照真实网关配置文件。

3. **UI/UX 交互设计**

   1. 布局结构
      - 整体为 **水平布局**，左侧「导入json」，右侧「抽取json」；
      - 每块内部再垂直布局：上方为按钮区，下方为代码块区；
      - 代码块为格式化文本，支持自动换行（`white-space: pre-wrap; word-break: break-all;`），后续可以增加按 key 查找。

   2. 左侧「导入json」
      - 按钮区：
        - 提供按钮 **「需要翻译」**。
      - 数据区：
        - 展示当前选中 JSON 配置文件的 Mock 内容：
          - 第一行：`// 导入配置文件（JSON）：<文件名>`；
          - 下方为若干 key/value（如 `login.title` 等）模拟真实配置。
        - 行为（原型简化版）：
          - 点击「需要翻译」后，通过 `setTimeout` 模拟抽取过程；
          - 完成后在右侧「抽取json」中补全一部分 key 的翻译内容，用于体现“被标记为需要翻译”。

   3. 右侧「抽取json」
      - 按钮区：
        - **「编辑」**（浏览态展示）：右侧代码块切换为可编辑 `<textarea>`；
        - **「保存」**（编辑态展示）：退出编辑态，仅保留当前内容，轻提示“已保存抽取json（原型，仅本地状态）”；
        - **「导入」**：模拟调用导入接口。
      - 数据区与状态：
        - 初始为与左侧结构对齐、value 为空字符串的 JSON；
        - 当左侧点击「需要翻译」时：
          - 右侧显示 loading（用 `<a-spin>` 包裹代码块），`setTimeout(800ms)` 后结束；
          - 部分 key 的 value 填充为 Mock 文本，模拟“已从导入 JSON 抽取到抽取 JSON”；
        - 当点击「导入」时：
          - 请求体包含：
            - `importJson`：左侧代码块完整文本；
            - `extractJson`：右侧当前 JSON 文本；
          - 使用 `setTimeout(1000ms)` 模拟接口耗时，仅在控制台输出请求体，不真实发请求。

> 说明：XML 配置文件导入仍然调用真实接口逻辑；JSON 子 Tab 走上述纯前端原型流程，用于验证交互体验。
