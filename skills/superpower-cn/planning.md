# 技能：制定计划 (Planning)

## 目标
将头脑风暴确定的方案拆解为具体的、可观察的开发步骤。

## 流程指令
1. **任务拆解**：将大需求拆分为 Task 1, Task 2... 每一项应足够小，以便单次提交。
2. **前后置依赖**：标明哪些步骤必须先完成（如：先定义接口，再写实现）。
3. **验证点**：为每个 Task 定义一个简单的验证方法（如何证明这步写对了？）。
4. **文件清单**：列出需要创建或修改的文件路径。

## 输出要求
- 采用任务列表形式（Task List）。
- **严禁**在该阶段写业务代码。
- 完成后询问：“需要我开始执行第一个任务吗？”
