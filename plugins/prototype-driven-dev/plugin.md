# 原型驱动开发插件

一套完整的原型驱动开发工作流，核心在于"以代码为中心的设计（Code-Centric Design）"。标准化人与AI的分工，提供可直接使用的技能集合。

## 功能特性

### 1. 完整工作流
- **S1. 极速原型** - 创建原型目录和硬编码Mock数据
- **S2. 体验测试** - 浏览器交互体验测试
- **S3. 逆向契约** - 生成OpenAPI规范
- **S4. 生成Mock** - 创建云端Mock服务
- **S5. 真实对接** - 实现HTTP请求的正式代码

### 2. 标准化分工
- 明确人与AI的分工
- 标准化数据流转
- 提高开发效率

### 3. 技能集合
- **Skill 1** - 原型创建
- **Skill 2** - API规范生成
- **Skill 3** - 真实对接开发

## 触发方式

### 推荐用法
- `使用原型驱动开发工作流`
- `开始原型驱动开发`
- `创建功能原型`

## 工作流阶段

### 阶段1：设计
- **S1. 极速原型** - Cursor执行
- 输入：自然语言描述需求 + 参考页面路径
- 输出：原型目录自包含 + 硬编码Mock数据

### 阶段2：验证
- **S2. 体验测试** - 人工执行
- 输入：浏览器交互体验
- 输出：修改意见 / 确认通过

### 阶段3：定义
- **S3. 逆向契约** - Cursor执行
- 输入：确认后的原型代码（含数据结构）
- 输出：OpenAPI规范

### 阶段4：托管
- **S4. 生成Mock** - Apifox工具
- 输入：OpenAPI规范文件
- 输出：云端Mock URL

### 阶段5：开发
- **S5. 真实对接** - Cursor执行
- 输入：云端Mock URL + 原型代码
- 输出：包含HTTP请求的正式代码
