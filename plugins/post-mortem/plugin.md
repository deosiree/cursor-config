# 开发复盘（Post-Mortem）插件

完成一个功能/模块开发后需要复盘，或用户要求总结经验、整理面经、归纳弯路时使用。

## 功能特性

### 1. 结构化文档生成
- **经验总结.md** - 四段式精炼总结核心内容
- **语法扫盲.md** - 涉及的技术概念和语法要点
- **弯路.md** - 踩坑与修正经验

### 2. 四段式经验总结
- **背景** - 业务痛点 + 工程问题（为什么要做）
- **实现** - 最终方案长什么样（做了什么）
- **核心** - 3-5个关键设计点（面试要讲清楚的）
- **取舍** - 工程权衡与边界（不做什么、为什么不做）

### 3. 知识沉淀
- 将开发经验转化为可复用的知识
- 创建团队知识库
- 支持新人培训和项目交接

## 触发方式

### 推荐用法
- `@post-mortem 总结这个功能的开发经验`
- `生成开发复盘文档`
- `整理这个模块的面经`

## 输出结构

```
docs/{开发主题}/
├── 经验总结.md   # 核心设计与决策
├── 语法扫盲.md   # 涉及的技术概念
└── 弯路.md       # 踩坑与修正
```

## 使用场景

### 场景1：功能开发完成
- 完成一个功能/模块开发后需要复盘
- 总结开发经验和教训
- 创建团队知识库

### 场景2：面试准备
- 整理技术面试材料
- 总结项目经验和设计思路
- 准备技术面经

### 场景3：团队培训
- 为新成员提供学习材料
- 分享开发经验和最佳实践
- 建立团队知识传承机制
