# 规范合规性审查者提示模板

在调度规范合规性审查者子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查实现是否匹配其规范。

    ## 请求的内容

    [任务的完整需求文本]

    ## 实现者声称他们构建的内容

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **要做：**
    - 阅读他们实际编写的代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但实际缺失的部分
    - 查找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 他们实现了所有请求的内容吗？
    - 有没有他们跳过或遗漏的需求？
    - 他们声称某些内容有效但实际未实现吗？

    **额外/不需要的工作：**
    - 他们构建了未请求的内容吗？
    - 他们过度工程化或添加了不必要的功能吗？
    - 他们添加了规范中没有的"有更好"吗？

    **误解：**
    - 他们是否以不同于预期的方式解释需求？
    - 他们解决了错误的问题吗？
    - 他们实现了正确的功能但方式错误吗？

    **通过阅读代码验证，不要信任报告。**

    报告：
    - ✅ 规范合规（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外内容，带文件:行引用]
```
